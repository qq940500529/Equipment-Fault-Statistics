# 测试目录

本目录用于存放测试文件和测试脚本。

## 测试策略

根据IMPLEMENTATION.md中的测试规范，本项目采用以下测试策略：

### 1. 单元测试
- 测试每个模块的独立功能
- 覆盖率目标：> 80%
- 测试框架：待选择（建议使用Jest或Mocha）

### 2. 集成测试
- 测试数据流的完整性
- 测试模块间的协作

### 3. 对比测试
- 与VBA处理结果进行对比
- 确保数据一致性

### 4. 性能测试
- 测试不同数据量的处理速度
- 目标：1000行数据 < 1秒

## 运行测试

```bash
# 安装测试依赖
npm install --save-dev jest

# 运行所有测试
npm test

# 运行特定测试
npm test -- <test-file>

# 生成覆盖率报告
npm test -- --coverage
```

## 测试用例编写规范

- 使用describe和it组织测试
- 每个测试应该独立运行
- 使用有意义的测试描述
- 测试边界情况和异常情况
- 保持测试简单和可维护
