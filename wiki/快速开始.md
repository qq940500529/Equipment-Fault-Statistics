# 快速开始

本指南将帮助您在5分钟内开始使用设备故障统计系统。

## 前置要求

- 现代浏览器（Chrome 90+, Firefox 88+, Safari 14+）
- Excel文件（.xlsx 或 .xls 格式）

## 三种使用方式

### 方式1: 直接运行（推荐）

这是最简单的方式，无需任何配置。

1. **下载项目**
   ```bash
   git clone https://github.com/qq940500529/Equipment-Fault-Statistics.git
   cd Equipment-Fault-Statistics
   ```

2. **打开应用**
   - 双击 `index.html` 文件
   - 或者在浏览器中打开该文件

3. **开始使用** 🎉

### 方式2: 本地Web服务器

如果直接打开HTML文件遇到问题，可以使用本地服务器。

**使用Python:**
```bash
# Python 3
python -m http.server 8000

# 访问 http://localhost:8000
```

**使用Node.js:**
```bash
# 使用npx (无需安装)
npx http-server

# 或使用全局安装
npm install -g http-server
http-server

# 访问 http://localhost:8080
```

### 方式3: 在线访问

访问GitHub Pages部署版本（需要项目启用GitHub Pages）：
```
https://qq940500529.github.io/Equipment-Fault-Statistics/
```

## 基本使用流程

### 步骤1: 上传Excel文件 📁

1. 点击上传区域或拖拽Excel文件
2. 系统支持 `.xlsx` 和 `.xls` 格式
3. 文件大小限制: 50MB

**Excel文件要求:**
- 必须包含以下列：
  - 工单号
  - 车间
  - 维修人
  - 报修时间
  - 维修开始时间
  - 维修结束时间

### 步骤2: 预览数据 📊

文件上传后，系统会自动：
- 解析Excel数据
- 显示数据统计信息
- 展示前50行数据预览
- 进行数据验证

**验证内容:**
- 必需列完整性检查
- 数据类型验证
- 日期格式验证
- 空值检测

### 步骤3: 处理数据 ⚙️

点击"开始处理"按钮，系统将自动：

1. **删除"合计"行**
   - 识别车间列值为"合计"的行
   - 自动删除

2. **车间列分列**
   - 将"一车间-A区"分割为
   - 车间: "一车间"
   - 区域: "A区"

3. **维修人分类**
   - 维修工: 16人
   - 电工: 14人
   - 未知: 其他人员

4. **时间计算**
   - 等待时间 = 维修开始 - 报修时间
   - 维修时间 = 维修结束 - 维修开始
   - 故障时间 = 等待时间 + 维修时间

5. **删除不完整行**
   - 删除时间数据缺失的行

### 步骤4: 查看结果 ✅

处理完成后，您可以：
- 查看处理统计信息
  - 原始行数
  - 处理后行数
  - 删除的合计行数
  - 时间不完整删除数
  - 有效数据率
- 预览处理后的数据

### 步骤5: 导出数据 💾

选择导出格式：

**Excel格式 (.xlsx)**
- 完整的Excel工作簿
- 保留格式和样式
- 适合进一步编辑

**CSV格式 (.csv)**
- 纯文本格式
- 兼容性最好
- 适合导入其他系统

**JSON格式 (.json)**
- 结构化数据
- 适合程序处理
- 便于数据交换

## 常见问题

### 文件上传失败？

**可能原因:**
- 文件格式不支持（只支持 .xlsx 和 .xls）
- 文件大小超过50MB
- 文件损坏或密码保护

**解决方法:**
- 检查文件格式
- 压缩数据或分批处理
- 使用无密码保护的文件

### 数据验证失败？

**可能原因:**
- 缺少必需列
- 列名不匹配
- 数据格式错误

**解决方法:**
- 检查Excel文件是否包含所有必需列
- 确保列名正确
- 参考 [数据结构](数据结构) 文档

### 处理结果不符合预期？

**可能原因:**
- 数据格式不标准
- 日期格式错误
- 维修人姓名不在名单中

**解决方法:**
- 查看 [数据处理流程](数据处理流程) 了解详细规则
- 检查 [维修人员名单](维修人员名单)
- 查看验证结果中的警告信息
